# Inventory Bot V10 — Система управления складом и активами

Веб-приложение на Flask для комплексного учёта активов, управления складом и контроля запасов.

---

## Возможности

### Учёт номенклатуры
- Каталог товаров с описаниями, единицами измерения, категориями и атрибутами
- Три метода учёта: **количественный**, **партионный** (с номерами партий и сроками годности), **поэкземплярный** (серийные номера, инвентарные номера)
- Загрузка и управление фотографиями товаров
- Вариации номенклатуры (разные артикулы одного товара)

### Складской учёт
- Многоуровневая структура: склады → места хранения → ячейки хранения
- Учёт остатков по складам и ячейкам
- Управление партиями (поставщик, дата, срок годности)
- Учёт отдельных экземпляров (серийные номера)
- Резервирование товаров с указанием срока резерва

### Документооборот
- Типы документов: поступления, выдачи, возвраты, перемещения, внутренние операции
- Статусы документов: черновик → проведён → архив
- Выдача оборудования сотрудникам с отслеживанием по серийным номерам
- Привязка расходов к центрам затрат и целям расходования

### Отчётность (15+ видов отчётов)
- Остатки на складах, предупреждения о низком запасе
- Движение товаров, анализ оборачиваемости
- Выдача сотрудникам, поступления от поставщиков
- Прибыль/убыток, неактивные позиции
- Экспорт всех отчётов в Excel/CSV

### Управление персоналом
- Справочники сотрудников и отделов
- Отслеживание закреплённого оборудования
- История выдачи по каждому сотруднику

### Системные функции
- Ролевая модель доступа (admin, user и др.)
- Полный журнал действий пользователей
- История входов с IP-адресами
- Инвентаризация (проведение, сверка, отчёт по расхождениям)
- Импорт номенклатуры из Excel
- Настраиваемые наименования полей (переводы)
- Уведомления, наборы (комплекты), автоматические правила категоризации

## Быстрый старт

### Windows (рекомендуется)

```bat
startV9.bat
```

Скрипт автоматически:
1. Проверит наличие Python
2. Установит зависимости из `requirements.txt`
3. Создаст нужные папки
4. Инициализирует базу данных при необходимости
5. Запустит приложение и откроет браузер

### Ручной запуск

```bash
# Создать и активировать виртуальное окружение
python -m venv venv
source venv/Scripts/activate   # Windows
# source venv/bin/activate     # Linux/macOS

# Установить зависимости
pip install -r requirements.txt

# Создать нужные папки
mkdir -p data static/uploads static/exports

# Запустить приложение
python app.py
```

**Адрес:** http://localhost:5000
**Логин по умолчанию:** `admin` / `admin123`

## Конфигурация

Файл `.env` в корне проекта:

```env
SECRET_KEY=your_secret_key_here
DATABASE_PATH=data/assets.db
DEBUG=True
```

Основные параметры в `config.py`:

| Параметр | Значение по умолчанию | Описание |
|---|---|---|
| `MAX_CONTENT_LENGTH` | 10 MB | Максимальный размер загружаемого файла |
| `PERMANENT_SESSION_LIFETIME` | 7 дней | Время жизни сессии |
| `REPORT_PAGE_SIZE` | 50 | Строк на страницу в отчётах |
| `MAX_EXPORT_ROWS` | 10 000 | Максимум строк при экспорте |
